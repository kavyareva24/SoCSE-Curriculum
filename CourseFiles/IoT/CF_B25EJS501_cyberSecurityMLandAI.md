## 1. Course description 🟢

This course introduces students to cybersecurity concepts and equips them with practical skills to use machine learning and artificial intelligence for threat detection, anomaly analysis, intrusion detection, and defence automation. Students learn how security data, logs, network traces, and malware behaviour can be represented and analysed using ML pipelines, while also understanding the ethical, adversarial, and explainability challenges involved in AI-driven security systems.

## 3. 🔵 Dual-level scope of each unit (KEYSTONE)

| Unit | Awareness level — every student must reach (the floor) | Advanced level — required to exceed 8 CGPA (the ceiling) |
|---|---|---|
| 1 | Explain the core cyber-security principles, common attacks, and the role of AI/ML in monitoring security data; identify basic threat indicators and describe the standard ML workflow. | Analyse the trade-off between feature choices and security-detection quality; justify data-labelling and evaluation choices for a novel threat dataset. |
| 2 | Apply logistic regression, decision trees, and random forest to instructor-provided security datasets; compute confusion-matrix metrics such as accuracy, precision, recall, and F1 score. | Compare multiple classifiers for an unseen security problem, tune hyperparameters, and justify the final model selection using ROC-AUC and error analysis. |
| 3 | Apply K-means or isolation forest to standard anomaly datasets and explain how anomaly scores support intrusion-detection decisions. | Design and evaluate an anomaly-detection pipeline for a new network or log dataset; critique model limitations and propose robust monitoring strategies. |
| 4 | Describe adversarial attacks, explainability, and defensive AI practices in security applications; interpret simple model explanations. | Design a secure AI-driven analysis workflow for a real case study, assess adversarial risks, and recommend explainability and privacy-preserving safeguards. |

## 4. Course objectives 🟢

- Understand foundational cyber-security concepts and the role of AI/ML in automated threat analysis.
- Apply machine-learning techniques to security datasets for phishing, malware, intrusion, and anomaly detection.
- Evaluate model performance using appropriate metrics and explain model decisions in a security context.
- Analyse adversarial AI threats and propose defensive and privacy-preserving strategies.
- Develop a mini-project that demonstrates an end-to-end security analytics pipeline using AI/ML tools.

## 8. 🔵 Differentiated instruction (mapped to the two levels)

**Reaching the floor (awareness level) — for students at risk of not clearing it:**
- Weekly remedial practice on core concepts such as attack types, metrics, and basic ML pipeline steps.
- Scaffolded lab worksheets with worked examples and guided step-by-step notebooks.
- Peer mentoring and short recap quizzes before IA and SEE.

**Reaching the ceiling (advanced level) — for students aiming above 8 CGPA:**
- Stretch tasks requiring model comparison, tuning, adversarial-risk analysis, and case-study critique.
- Optional reading from NIST, MITRE ATT&CK, and explainability papers to deepen security-AI understanding.
- Advanced mini-project variants that require design choices, evaluation, and evidence-based justification.

## 22. 🔵 Implementation strategy — Merrill's First Principles

This course uses Merrill’s First Principles of Instruction to make learning active and transferable. Students begin with a problem or task, activate prior knowledge, observe a demonstration, practise in guided exercises and labs, and finally integrate the skill into a mini-project or novel-security case.
