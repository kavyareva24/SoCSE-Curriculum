## 1. Course description 🟢

This course introduces students to the C# programming language and the .NET ecosystem, with emphasis on modern .NET Core concepts, object-oriented programming, exception handling, interfaces, and application development practices. Students will learn to write, debug, and structure programs using C# while gaining familiarity with the .NET runtime, assemblies, types, and the common language infrastructure.

---

## 2. Course content (units & weightage) 🟢

| Unit | Syllabus | Weightage |
|---|---|---|
| 1 | Introducing C# and .NET (Core) 5; overview of the .NET platform, runtime, CTS, CLS, assemblies, namespaces, and building .NET Core applications | 25 Marks |
| 2 | Core C# programming constructs — data types, strings, conversions, variables, arrays, methods, enum, structures, nullable types, tuples, and iteration constructs | 25 Marks |
| 3 | Object-oriented programming with C# — encapsulation, constructors, this/static keywords, access modifiers, automatic properties, object initialization, partial classes, records, inheritance, polymorphism, and System.Object | 25 Marks |
| 4 | Exception handling, object lifetime, garbage collection, finalizers, IDisposable, lazy object instantiation, interfaces, interface hierarchies, IEnumerable/IEnumerator, ICloneable, IComparable | 25 Marks |

## 3. 🔵 Dual-level scope of each unit (KEYSTONE)

| Unit | Awareness level — every student must reach (floor) | Advanced level — required to exceed 8 CGPA (ceiling) |
|---|---|---|
| 1 | Explain the basics of .NET Core, assemblies, namespaces, CTS/CLS, and build a simple C# application using Visual Studio or VS Code | Compare .NET runtime components, evaluate .NET support lifecycle implications, and explain how assemblies and the CLR support application execution |

| 4 | Handle common exceptions, implement basic interfaces, and explain object lifetime and garbage collection | Design robust applications using custom exception handling, interface-based abstractions, and disposal/lazy initialization patterns |

## 4. Course objectives 🟢

- Introduce the architecture of the .NET platform and the C# programming model.
- Develop problem-solving skills using core C# language constructs and object-oriented principles.
- Enable students to build modest console or desktop applications with structured code and reusable components.
- Promote robust application design through exception handling, interfaces, and resource management.

## 8. 🔵 Differentiated instruction (mapped to the two levels)

**Reaching the floor (awareness level) — for students at risk of not clearing it:**
- Stepwise guided coding demonstrations with starter templates and syntax checklists.
- Short recap sessions on basic syntax, control flow, and OOP structures.
- Remedial problem sheets focused on writing and debugging simple C# programs.

**Reaching the ceiling (advanced level) — for students aiming above 8 CGPA:**
- Stretch tasks requiring interface-based design, reusable class hierarchies, and robust exception handling.
- Advanced mini-projects that demonstrate polymorphism, resource management, and clean architectural design.
- Optional reading from Microsoft documentation and contemporary .NET programming patterns.

## 22. 🔵 Implementation strategy — Merrill's First Principles

Merrill's First Principles are used to structure the course around activation, demonstration, application, and integration. In this course, the awareness level is usually delivered through conventional explanation and guided practice, while the advanced ceiling is reached through stretch problems, design-oriented tasks, and small integrated applications.
