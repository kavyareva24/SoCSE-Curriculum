## 1. Course description 🟢

This course introduces students to modern web design and development by combining responsive front-end UI construction with lightweight server-side workflows. Learners will build HTML, CSS, JavaScript, and Node.js/Express skills through short in-class activities, quizzes, and mini-project checkpoints.

---

## 2. Course content (units & weightage) 🟢

| Unit | Syllabus | Weightage |
|---|---|---|
| 1 | Web application fundamentals, HTML semantics, CSS styling, responsive layout, and accessibility | 25 Marks |
| 2 | JavaScript programming, DOM interaction, event handling, and client-side validation | 25 Marks |
| 3 | Node.js/Express server-side design, RESTful routing, middleware and API handling | 25 Marks |
| 4 | Database integration, CRUD operations, deployment, security and full-stack project integration | 25 Marks |

---

## 3. 🔵 Dual-level scope of each unit (KEYSTONE)

| Unit | Awareness level — every student must reach (floor) | Advanced level — required to exceed 8 CGPA (ceiling) |
|---|---|---|
| 1 | Build semantic HTML pages, apply CSS styling, and implement a responsive layout for common screen sizes | Design accessible, production-aware page structures, advanced layout patterns, and responsive behaviour across diverse devices |
| 2 | Write JavaScript to control page behaviour, manipulate the DOM, handle events, and validate user inputs | Create modular client-side logic, manage UI state, and implement asynchronous interactivity for richer user experience |
| 3 | Build an Express server with routes, JSON responses, and middleware for input parsing | Design secure REST APIs with structured request/response flows, authentication checks, and error handling for maintainability |
| 4 | Connect front end to back end, perform CRUD operations, deploy a working app, and apply basic security checks | Integrate database-backed persistence, deployment automation, performance tuning, and security hardening for a production-ready app |

- Introduce the development workflow for building responsive web applications.

## 8. 🔵 Differentiated instruction (mapped to the two levels)

**Reaching the floor (awareness level) — for students at risk:**
- Provide starter templates and guided code snippets for each in-class task.
- Use checklists for HTML structure, CSS layout, JavaScript event binding, and API flow.
- Offer remedial lab sessions and quick recovery exercises after IA performance.

**Reaching the ceiling (advanced level) — for students aiming above 8 CGPA:**
- Offer stretch tasks such as reusable UI modules, asynchronous state updates, and deployment automation.
- Invite students to implement role-based UI behaviour, caching, or security headers.
- Assign advanced review tasks on API design, performance, and security hardening.
