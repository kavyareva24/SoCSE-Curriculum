## 1. Course description 🟢

This course introduces the fundamental concepts of data structures and algorithms with an emphasis on efficient problem solving. Students learn to represent, organize, and manipulate data using standard structures and to evaluate algorithmic trade-offs for correctness, time, and space complexity.

---

## 2. Course content (units & weightage) 🟢

| Unit | Syllabus | Weightage |
|---|---|---|
| 1 | Introduction to data structures, complexity analysis, arrays, linked lists, stacks, queues, recursion | 25 Marks |
| 2 | Trees, binary search trees, balanced trees, heaps, hashing, collision handling | 25 Marks |
| 3 | Sorting and searching techniques, graphs, graph traversal, shortest path algorithms | 25 Marks |
| 4 | Algorithm design strategies: greedy, divide and conquer, dynamic programming, applications and case studies | 25 Marks |

---

## 3. 🔵 Dual-level scope of each unit (KEYSTONE)

| Unit | Awareness level — every student must reach (floor) | Advanced level — required to exceed 8 CGPA (ceiling) |
|---|---|---|
| 1 | Understand the purpose of common linear data structures, implement them correctly, and solve standard textbook problems using the standard procedure | Analyze trade-offs among arrays and linked representations, optimize implementations, and apply them to unfamiliar or larger-scale problems |
| 2 | Explain tree and hashing concepts, perform insertion/deletion/search on standard structures, and interpret basic complexity | Design balanced/efficient tree-based or hashing-based solutions, compare alternatives, and justify design choices for performance-critical tasks |
| 3 | Apply standard sorting/searching and graph traversal methods to familiar problems | Evaluate algorithm choice for real-world graphs, compare performance, and extend traversal/shortest-path approaches to non-trivial scenarios |
| 4 | Recognize greedy, divide-and-conquer, and dynamic-programming patterns and solve routine examples | Formulate a suitable strategy for a novel problem, justify the approach, and optimize the solution for constraints and scalability |

---

## 4. Course objectives 🟢

- Introduce the fundamental concepts of data abstraction, data structures, and algorithmic thinking.
- Develop the ability to select appropriate data structures for solving computational problems.
- Build competence in analyzing algorithms for efficiency and correctness.
- Strengthen programming skills through implementation and application of structured problem-solving methods.

The course will be delivered through direct instruction, problem-based learning, guided programming practice, flipped classroom support, and collaborative discussion of algorithmic strategies.

## 8. 🔵 Differentiated instruction (mapped to the two levels)

**Reaching the floor (awareness level) — for students at risk:**
- Weekly remedial practice on fundamental concepts and implementation templates
- Short worked examples, screencasts, and guided problem sets for core operations
- Peer mentoring and scaffolded assignments with stepwise checkpoints

**Reaching the ceiling (advanced level) — for students aiming above 8 CGPA:**
- Advanced problem-solving tasks involving larger inputs, optimization, and comparative analysis
- Short reading assignments on algorithm design patterns and complexity trade-offs
- Optional stretch problems requiring design justification and implementation refinement
